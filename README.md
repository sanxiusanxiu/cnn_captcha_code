#### 项目总览

这是一个基于 CNN 的验证码识别模型，核心思路是将 4 位验证码（数字 0-9 + 大写字母 A-Z，共 36 种字符）识别建模为一个多标签分类问题，每张图输出一个 144 维向量（4 个位置 × 每个位置 36 类）

项目来源：https://github.com/Python3WebSpider/DeepLearningImageCaptcha

项目参考：https://github.com/dee1024/pytorch-captcha-recognition 

#### 数据准备

训练数据包含两部分，一部分是验证码图片，另一部分是验证码的真实标注，我们可以使用验证码生成器自行生成一些验证码，这样就同时有了验证码图片和标注数据

生成验证码的过程分为两步，第一步是生成随机的文本数据，第二步是根据生成的文本数据生成验证码图片

#### 搭建模型

整体结构：3 层卷积（提取图像特征） + 2 层全连接（分类输出）
输入：1 * 60 * 160 的灰度验证码图片
输出：144 维向量（4 个字符位置 * 每个位置 36 种字符的概率）
